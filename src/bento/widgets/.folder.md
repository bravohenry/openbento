# Folder: /src/bento/widgets

1. **"Status"**: Implementation layer for specific card types, encapsulating business logic as reusable Widget components, each Widget corresponds to a card type.
2. **"Logic"**: Receives configuration data, renders through BentoCard, implements specific business logic (such as link navigation, image display, map embedding, etc.).
3. **"Constraints"**: All Widgets must be registered in the registry, must be built on BentoCard, direct DOM manipulation is prohibited.
